Base

  • git config --global user.name blabla
    Встановлює ім'я, яке буде відображатися в поле автора у виконуваних ним комітах
  • git config --global user.email blabal@bla
    Встановлює адресу електронної пошти, яка буде відображатися в інформації про виконувані коміти
  • git init
    Створює новий локальний репозиторій з заданим ім'ям
  • git status
    Перераховує всі нові або змінені файли, які потребують фіксації
  • git add
    Індексує вказаний файл для подальших комітів
  • git commit
    Фіксує проіндексовані зміни і зберігає їх в історію версій
  • git log
    Історія комітів для поточної гілки
  • git checkout
    Перемикається на обрану гілку і оновлює робочу директорію до її стану
  • git branch
    Створює нову гілку
  • git push
    Використовується для вивантаження вмісту локального сховища в віддалений репозиторій.
  • git pull
    Завантажує історію з віддаленого сховища та об'єднує її з локальної
  • git merge
    Вносить зміни з гілки віддаленого сховища в поточну гілку локального сховища
  • git clone
    Викачує репозиторій разом з усією його історією змін

Gitignore

Gitignore це файл який завжди зберігається в проекті. Він містить правила виключення, кожен з яких має відносний шлях. Він додається і зберігається в самому репозиторії. Таким чином, будь-який інший розробник,який викачав собі проект матиме ці правила і не зробить помилку, додавши непотрібні файли.

Advanced

  • git revert - За допомогю цієї команди можна скасувати операцію без втрати історії коммітов
  • git reset - За допомогю цієї команди можна скасувати операцію
  • git rebase - Переміщує історію всієї гілки поверх іншої, переписуючи історію проекту новими коммітов.
  • git cherry-pick - Використовується для перенесення окремих коммітов з одного місця сховища в інше
  • git stash - Бере зміни, підготовлені і непідготовлені до фіксації, зберігає їх для подальшого використання і прибирає з робочою області.
